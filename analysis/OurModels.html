This parameter is called EXHAUSTIVE_ITERATION and it's placed in config.py. We tried different numerical values in exhaustive_iterations for every model and the results are shown next.

HYPERPARAMETERS OF THE MODELS

gbm_hyperparams = {

    'n_estimators': 2000,        
    'max_depth': 5,
    'learning_rate': 0.1,
    'loss': 'huber',
    'criterion': 'mse',
    'validation_fraction': 0.01,
    'n_iter_no_change': 20
}

That are the parameters we played with to get the best r squared. First of all we're going to explain what is every parameter. The first parameter 'n_estimators' means the number of estimators that we're going to use. 'Max depth' parameter has to be between 1 and 10 because if it's not in that range it's going to give us a negative r squared. 'learning_rate' is the parameter that shows us how our model is going to learn from data. It has a numerical value that in this case has to be 0.1 because if you increases this number the r squared gives negative value too. 'loss' means the loss function we use. There are 5 types of loss and we played with them and we declared that the best one was huber, which is the combination between 2 'loss' types. 'criterion' is what we want to maximize and in our case what we want to maximize is the r squared error so we type mse. There are 2 more types of 'criterion' but that was which gives us a better performance. And finally we have 'n_iter_no_change' and 'validation_fraction' that are parameters that goes together, n_iter_no_change is a parameter that looks what is the error and if the error don't change in the number of iterations that we specify then it stops.
